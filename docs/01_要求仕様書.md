# ブレインダンプアプリ 要求仕様書

## 1. 文書概要

本書は、Supabase をバックエンドとして利用する**自分専用のメモアプリ（ブレインダンプアプリ）**に対する要求を定義する。

## 2. 目的・背景

思考やアイデアを気軽に記録し、あとから見返せること。複数デバイスから同じデータにアクセスできること。

## 3. スコープ

| 項目 | 内容 |
|------|------|
| 対象ユーザー | 認証済みの各利用者（**自分のメモのみ**を扱う） |
| プラットフォーム | Web（スマートフォン・PC のブラウザ） |
| バックエンド | Supabase（認証・データベース・API） |
| 認証 | **メールアドレス・パスワードでのログイン機能を搭載している**（新規登録・ログイン・ログアウト・ログイン中のメールアドレス表示） |

本書は**機能要求**および**データの保全方針**を扱う。画面デザインの詳細やフロント実装スタックは実装フェーズで定める。

## 4. 用語定義

| 用語 | 意味 |
|------|------|
| メモ / ダンプ | ユーザーが入力したテキストの単位として保存される 1 件のレコード |
| 自分専用（メモ） | **ログインした利用者のみ**がメモ機能にアクセスでき、**自分が作成したメモのみ**を参照・追加・編集・削除できること（`brain_dumps.user_id` による所有者の紐付け。詳細は DB 仕様書・機能仕様書）。 |
| ログインユーザー | Supabase Auth で認証されたアカウント。メモ行の `user_id` は当該アカウントのユーザー ID（UUID）と一致する。 |

## 5. 機能要求

### 5.1 メモの入力・保存（FR-01）

- ユーザーはテキストでメモを入力し、保存できる。
- メモには**タイトル**と**本文**の両方が必須である。
- 保存時には**作成日時**がサーバー側で自動的に設定される。

### 5.2 メモ一覧（FR-02）

- ユーザーは保存されたメモを一覧として閲覧できる。
- 一覧には少なくとも識別に必要な情報（例: タイトル、作成日時、カテゴリ名（設定されている場合）など）が含まれること。本文の一覧表示可否は UX により決定してよい。

### 5.3 メモの編集（FR-03）

- ユーザーは既存メモを編集し、内容を更新できる。
- 編集可能な項目は実装および DB 定義に従う（最低限タイトル・本文。カテゴリは FR-07 に従い任意）。

### 5.4 メモの削除（FR-04）

- ユーザーは既存メモを削除できる。
- 削除後、当該メモは一覧・参照対象から除かれること。

### 5.5 認証・アクセス制御（FR-05）

- **メールアドレスとパスワード**による認証のみとする（OAuth・マジックリンクは対象外）。
- 提供する認証機能は次の **4 つに限定**する：**新規登録**、**ログイン**、**ログアウト**、**ログイン中のメールアドレス表示**。
- **メール確認（confirm email）は行わない**（Supabase 側でもメール確認を必須としない運用とする）。
- パスワードは **6 文字以上**とする。
- 新規登録またはログインに成功したら、**同一 `index.html` 内でメモ UI を表示**する（別ファイルへの遷移は行わない）。
- 認証処理のエラーは **`alert()`** でユーザーに通知してよい。
- 未ログインユーザーは、メモの閲覧・作成・編集・削除などの主要機能へのアクセスができないようにする（未ログイン時は **`index.html` 上の認証 UI** を表示し、メモ UI は非表示とする）。

### 5.6 メモのユーザー別管理（FR-08）

- **ユーザーはログイン後、自分が作成したメモのみを参照・追加できる。**
- 新規保存時、当該メモ行には **ログイン中ユーザーの ID** を `user_id` として記録する（DB 仕様書参照）。
- 一覧・編集・削除は、**`user_id` がログイン中ユーザーと一致する行**に限定する。他ユーザーのメモは一覧に表示せず、更新・削除もできないこと。
- カテゴリマスタ（`memo_categories`）は引き続き **全認証ユーザーが参照できる共有の選択肢**として読み込んでよい（ユーザー別マスタの新規作成・編集は本要件のスコープ外）。

### 5.9 DB レベルのアクセス制御（FR-09）

- **ログイン済みユーザーは自分が作成したデータのみ操作できる**（メモに限る。カテゴリマスタは §5.8・RLS 仕様書のとおり参照のみ）。
- Supabase で対象テーブルに **RLS を有効化**し、**`authenticated` ロール**に対してポリシーを定義する（**`anon` は原則としてデータ操作不可**）。
- メモテーブル **`brain_dumps`** では、**SELECT / INSERT / UPDATE / DELETE** すべてにおいて **`user_id = auth.uid()`** を満たす行のみを許可する。
- カテゴリテーブル **`memo_categories`（画面上の categories）** では、認証済みユーザーに **SELECT のみ**許可し、**INSERT / UPDATE / DELETE はクライアントに許可しない**。
- RLS 用 SQL の生成・適用は別タスクとし、本要求では **`docs/05_RLS仕様書.md`** にポリシー内容を定義する。

### 5.7 レスポンシブ / マルチデバイス（FR-06）

- スマートフォンおよび PC のブラウザから利用可能な UI とする（レイアウトが画面幅に適応すること）。
- 同一アカウントでデバイスをまたいで同じデータにアクセスできること。

### 5.8 カテゴリ管理（FR-07）

- メモにカテゴリを設定できる。
- カテゴリの選択肢は **専用のマスタテーブル**で管理する（名称・並び順等の詳細は DB 仕様書）。
- メモの投稿および編集時に、当該マスタ由来の一覧をドロップダウンなどで選択できる。
- カテゴリは**必須項目としない**（未選択のメモが存在してよい）。
- メモ一覧画面に、そのメモが紐づくカテゴリ名が表示される（未設定時の表示文言は UX にて決定してよい）。

## 6. 非機能要求

### 6.1 セキュリティ

- 通信は HTTPS を前提とする（Supabase 標準）。
- **ログイン済みユーザーは自分が作成したデータのみ操作できる。** これを DB 層でも担保するため、Supabase の **Row Level Security（RLS）** を **`brain_dumps` およびカテゴリ用テーブル（`memo_categories`）** に適用する（ポリシーの詳細は **`docs/05_RLS仕様書.md`**）。
- メモ（`brain_dumps`）は **`user_id` と `auth.uid()` の一致**により、参照・追加・更新・削除の対象を本人の行に限定する。
- カテゴリマスタはアプリから **参照（SELECT）のみ**行い、**ユーザーごとの作成・更新・削除は行わない**（RLS 上もクライアントからの書き込みは許可しない方針とする。詳細は RLS 仕様書）。
- アプリ側のクエリ条件（`user_id` 絞り込み等）と **RLS は併用**し、クライアントの実装漏れがあっても DB で他ユーザーのメモに触れないようにする。

### 6.2 可用性・運用

- Supabase の提供する SLA・制限に従う。
- アプリ側でバックアップ要件がある場合は別途運用設計する（本書では必須としない）。

## 7. 前提・制約

- データは **`brain_dumps`（メモ）** を中心とし、カテゴリ選択肢は **専用のマスタテーブル** で管理する（テーブル名・関連は DB 仕様書）。
- **`brain_dumps` に `user_id` 列を追加する**（メモの所有者を Supabase Auth のユーザー ID で保持する）。カラム追加以外のスキーマ変更は RLS 適用に必要な範囲に限る（`memo_categories` への `user_id` 追加は行わない）。
- **RLS** は `brain_dumps` と `memo_categories` に対して **`docs/05_RLS仕様書.md`** に従い設定する。

## 8. 受入基準（概要）

| ID | 内容 |
|----|------|
| AC-01 | ログイン後、メモを保存でき、一覧に反映される |
| AC-02 | 一覧からメモを開き編集できる |
| AC-03 | メモを削除でき、一覧から消える |
| AC-04 | 未ログイン状態ではメモの閲覧・操作ができない（`index.html` で認証 UI のみ表示） |
| AC-05 | 投稿・編集でマスタ由来のドロップダウンからカテゴリを選べる（未選択でも保存できる）。一覧でカテゴリ名が分かる |
| AC-06 | メール・パスワード（6文字以上）で新規登録でき、成功後 `index.html` でメモが使える |
| AC-07 | ログイン・ログアウトができ、ログイン中は画面上にメールアドレスが表示される |
| AC-08 | ログアウト後はメモ画面にアクセスできない |
| AC-09 | ユーザー A で保存したメモは、ユーザー B の一覧に表示されない |
| AC-10 | ログイン中ユーザーが新規保存したメモは、自分の一覧にのみ表示される |
| AC-11 | RLS 適用後、他ユーザーの `brain_dumps` 行を API 経由で SELECT / UPDATE / DELETE できない |
| AC-12 | 未ログイン（`anon`）では `brain_dumps` / `memo_categories` へアクセスできない |
| AC-13 | ログイン済みユーザーは `memo_categories` を SELECT できるが、クライアントから INSERT / UPDATE / DELETE できない |
