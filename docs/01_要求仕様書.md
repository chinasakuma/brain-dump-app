# ブレインダンプアプリ 要求仕様書

## 1. 文書概要

本書は、Supabase をバックエンドとして利用する**自分専用のメモアプリ（ブレインダンプアプリ）**に対する要求を定義する。

## 2. 目的・背景

思考やアイデアを気軽に記録し、あとから見返せること。複数デバイスから同じデータにアクセスできること。

## 3. スコープ

| 項目 | 内容 |
|------|------|
| 対象ユーザー | アプリ運用者本人（自分専用の利用を想定） |
| プラットフォーム | Web（スマートフォン・PC のブラウザ） |
| バックエンド | Supabase（認証・データベース・API） |

本書は**機能要求**および**データの保全方針**を扱う。画面デザインの詳細やフロント実装スタックは実装フェーズで定める。

## 4. 用語定義

| 用語 | 意味 |
|------|------|
| メモ / ダンプ | ユーザーが入力したテキストの単位として保存される 1 件のレコード |
| 自分専用 | 運用上は単一利用者向けとし、**ログインした利用者のみ**がメモ機能にアクセスできること。DB 上は `user_id` による行の分離は行わない（詳細は DB 仕様書）。 |

## 5. 機能要求

### 5.1 メモの入力・保存（FR-01）

- ユーザーはテキストでメモを入力し、保存できる。
- メモには**タイトル**と**本文**の両方が必須である。
- 保存時には**作成日時**がサーバー側で自動的に設定される。

### 5.2 メモ一覧（FR-02）

- ユーザーは保存されたメモを一覧として閲覧できる。
- 一覧には少なくとも識別に必要な情報（例: タイトル、作成日時、カテゴリ名（設定されている場合）など）が含まれること。本文の一覧表示可否は UX により決定してよい。

### 5.3 メモの編集（FR-03）

- ユーザーは既存メモを編集し、内容を更新できる。
- 編集可能な項目は実装および DB 定義に従う（最低限タイトル・本文。カテゴリは FR-07 に従い任意）。

### 5.4 メモの削除（FR-04）

- ユーザーは既存メモを削除できる。
- 削除後、当該メモは一覧・参照対象から除かれること。

### 5.5 認証・アクセス制御（FR-05）

- 未ログインユーザーは、メモの閲覧・作成・編集・削除などの主要機能へのアクセスができないようにする。
- メモデータは **`user_id` で行分離しない**ため、同一プロジェクトに複数の認証ユーザーが存在する環境では、理論上はログイン済みユーザー間でデータが共用される。この前提を運用・公開範囲で管理する。**アカウント単位でメモを厳密に分離する要件は本仕様のスコープ外**とする（必要な場合は DB に所有者列と RLS を導入する別要件とする）。

### 5.6 レスポンシブ / マルチデバイス（FR-06）

- スマートフォンおよび PC のブラウザから利用可能な UI とする（レイアウトが画面幅に適応すること）。
- 同一アカウントでデバイスをまたいで同じデータにアクセスできること。

### 5.7 カテゴリ管理（FR-07）

- メモにカテゴリを設定できる。
- カテゴリの選択肢は **専用のマスタテーブル**で管理する（名称・並び順等の詳細は DB 仕様書）。
- メモの投稿および編集時に、当該マスタ由来の一覧をドロップダウンなどで選択できる。
- カテゴリは**必須項目としない**（未選択のメモが存在してよい）。
- メモ一覧画面に、そのメモが紐づくカテゴリ名が表示される（未設定時の表示文言は UX にて決定してよい）。

## 6. 非機能要求

### 6.1 セキュリティ

- 通信は HTTPS を前提とする（Supabase 標準）。
- DB 層の RLS は当面必須としない。機密性が高い場合は RLS やキー管理を別途検討する（DB 仕様書参照）。

### 6.2 可用性・運用

- Supabase の提供する SLA・制限に従う。
- アプリ側でバックアップ要件がある場合は別途運用設計する（本書では必須としない）。

## 7. 前提・制約

- データは **`brain_dumps`（メモ）** を中心とし、カテゴリ選択肢は **専用のマスタテーブル** で管理する（テーブル名・関連は DB 仕様書）。
- **`user_id` 列は持たない**（Table Editor での手入力負荷とスキーマ簡素化のため）。

## 8. 受入基準（概要）

| ID | 内容 |
|----|------|
| AC-01 | ログイン後、メモを保存でき、一覧に反映される |
| AC-02 | 一覧からメモを開き編集できる |
| AC-03 | メモを削除でき、一覧から消える |
| AC-04 | 未ログイン状態ではメモの閲覧・操作ができない |
| AC-05 | 投稿・編集でマスタ由来のドロップダウンからカテゴリを選べる（未選択でも保存できる）。一覧でカテゴリ名が分かる |
